---
name: wilco-grill
description: Interview the user relentlessly about a plan, PRD, or design until reaching shared understanding, resolving each branch of the decision tree one by one. Use when the user wants to stress-test a plan, get grilled on their design, review remaining ambiguities, or mentions "grill me".
---

Interview the user relentlessly about every aspect of the current plan, PRD, or design until you reach a shared understanding. Walk down each branch of the decision tree, resolve dependencies between decisions one by one, and give your recommended answer with each question.

Ask one question at a time.

If a question can be answered by exploring the codebase or existing docs, explore first instead of asking.

When the discussion is about repository-native planning, prefer alignment with:

- `docs/prd/active/`
- `docs/plans/active/`
- current architecture docs, if present

If the user is grilling a PRD or plan generated by other Wilco skills, keep the feedback compatible with `wilco-prd`, `wilco-plan`, and `wilco-docs`.
