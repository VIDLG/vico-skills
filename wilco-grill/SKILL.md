---
name: wilco-grill
description: Interview the user relentlessly about a plan, PRD, or design until reaching shared understanding, resolving each branch of the decision tree one by one. Use when the user wants to stress-test a plan, get grilled on their design, review remaining ambiguities, or mentions "grill me".
---

Preserve the full core behavior of a strong grilling workflow:

- Interview the user relentlessly about every aspect of the current plan, PRD, or design until you reach a shared understanding.
- Walk down each branch of the decision tree, resolve dependencies between decisions one by one, and do not stop at superficial agreement.
- Give your recommended answer with each question.
- Ask one question at a time.
- If a question can be answered by exploring the codebase or existing docs, explore first instead of asking.

Maintain high-intensity questioning. Do not stop at surface agreement, and do not trade away necessary rigor just to be shorter or softer. The goal is to reach real clarity while staying aligned with Wilco's repository-native planning flow.

When the discussion is about repository-native planning, prefer alignment with:

- `docs/prd/active/`
- `docs/plans/active/`
- current architecture docs, if present

If the user is grilling a PRD or plan generated by other Wilco skills, keep the feedback compatible with `wilco-prd`, `wilco-plan`, and `wilco-docs`.

When a question exposes an unresolved issue, keep track of which document should absorb the resolution:

- PRD-level scope, goals, non-goals, or user stories -> `docs/prd/active/...`
- execution sequencing, acceptance criteria, or migration slicing -> `docs/plans/active/...`
- durable current truth that should outlive the plan -> `docs/architecture/...`

At the end of a substantial grilling pass, summarize:

- unresolved decisions
- your recommended resolutions
- where each accepted resolution should be written back
